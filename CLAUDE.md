코딩 시 아래 원칙을 반드시 지켜주세요.

1. 주석을 충분히 달아주세요.
- 주석은 유치원생도 이해할 수 있을 정도로 아주 쉽게 설명해주세요.
- “무슨 코드인지”뿐 아니라 “왜 이렇게 작성했는지”도 설명해주세요.

2. 성급하게 코드를 전체 수정하지 말아주세요.
- 먼저 현재 코드 구조와 흐름을 분석한 뒤 수정해주세요.
- 문제 원인과 수정 방향을 먼저 짧게 설명한 후 코드를 수정해주세요.
- 동작 중인 기존 기능은 최대한 유지하고, 필요한 부분만 최소 수정해주세요.

3. 유지보수가 쉬운 코드로 작성해주세요.
- 이해하기 쉬운 변수명, 함수명, 컴포넌트명을 사용해주세요.
- 복잡한 로직은 여러 단계로 나누고, 한 함수에 역할을 너무 많이 넣지 말아주세요.
- 하드코딩은 최소화하고, 반복되는 값은 상수나 재사용 가능한 구조로 분리해주세요.

4. 안전한 코드로 작성해주세요.
- null, undefined, 빈 값, 예외 상황을 고려해주세요.
- 임시 코드, 불필요한 console.log, 사용하지 않는 import는 최종 코드에 남기지 말아주세요.

5. 기존 프로젝트 스타일을 존중해주세요.
- 파일 전체를 불필요하게 재작성하지 말아주세요.
- 기존 네이밍, 구조, UI 스타일, 컴포넌트 패턴을 가능한 한 유지해주세요.
- 새 라이브러리 도입은 꼭 필요한 경우에만 해주세요.

6. 결과는 초보 개발자도 따라갈 수 있게 설명해주세요.
- 어떤 파일을 왜 수정했는지,
- 수정 전/후 핵심 차이가 무엇인지,
- 내가 다음에 무엇을 확인하면 되는지
짧고 명확하게 정리해주세요.

추측으로 수정하지 말고, 현재 코드 기준으로 확인 가능한 내용만 바탕으로 작업해주세요.
불확실한 부분은 임의로 결정하지 말고 가정을 명시해주세요.
작은 수정으로 해결 가능한 경우, 구조 개선이나 리팩토링을 이유로 범위를 키우지 말아주세요.
